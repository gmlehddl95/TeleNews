→ SequenceMatcher 유사도 계산
→ 🔥🔥(5건+) / 🔥(2-4건) / ⭐(단독) 자동 판별

💬 핵심 프롬프트 #3

그리고 앞으로 더 하락하면 TQQQ가 얼마나 떨어질지 예상가도 보여줘"

→ yfinance로 실시간 주가 추적
→ 2022년 폭락 데이터 기반 레버리지 계산 알고리즘

💬 핵심 프롬프트 #4

👤 "Render에 배포하려는데 계속 멈춰.

💻 핵심 구현 #1: 유사뉴스 필터링

SequenceMatcher 기반 유사도 계산

1️⃣ 제목에서 HTML 태그 제거
2️⃣ 특수문자 정규화
3️⃣ 유사도 계산 (0.0 ~ 1.0)
4️⃣ 0.5 이상이면 중복으로 판단
5️⃣ 대표 뉴스의 similar_count 증가

결과: "삼성전자 주가 상승" ≈ "삼성전자 주가 급등"

→ 하나만 전송, [관련뉴스: 2건] 표시

💻 핵심 구현 #2: TQQQ 예상가 계산

2022년 실제 폭락 데이터 기반 레버리지 배수

• 나스닥 20% 하락 → TQQQ 53% 하락 (2.60배)
• 나스닥 30% 하락 → TQQQ 75% 하락 (2.35배)
• 나스닥 40% 하락 → TQQQ 82% 하락 (2.15배)

현재가에서 목표가까지 변화율 × 레버리지 배수
→ 예상 TQQQ 가격 산출

변동성 손실(Decay) 반영으로 정확도 향상

🔧 문제 해결 사례

👤 "봇이 자꾸 멈춰요"

🤖 yfinance API가 무한 대기 → 30초 타임아웃 + 재시도 3회 적용

👤 "Render에서 데이터가 날아가요"

🤖 Ephemeral 파일시스템 → Railway PostgreSQL로 영구 보존

👤 "웹 서버는 15분 후 슬립 상태로 바뀌잖아?"

🤖 UptimeRobot이 5분마다 핑 → 깨어있는 상태 유지

👤 "현재 시간을 9시간 늦게 인식해요"

🤖 UTC 기준 → timezone(timedelta(hours=9))로 KST 적용
